# Urban Notes — Project Context

## What it is
A Go-based markdown note-taking web app. Single binary, no database — each note is a JSON file in `notes/`. Served at `http://localhost:8080`.

## Build & run
```bash
/home/murbanek/go/bin/go build -ldflags="-X main.version=v1.1.0" -o markdown-notes .
pkill -f markdown-notes 2>/dev/null; sleep 0.3
AUTH_USER=admin AUTH_PASS=***** ./markdown-notes &
```
`go` is at `/home/murbanek/go/bin/go` (not on PATH by default).
Exit code 144 from `pkill` is normal when the process wasn't running.

## File layout
```
main.go                  — Go HTTP server, JSON file storage, /api/render endpoint
static/
  index.html             — Single-page app shell
  app.js                 — All frontend logic (~1500 lines, no build step)
  style.css              — All styles (design tokens + components)
  favicon.ico / *.png    — Icons generated by /tmp/gen_icons.py (Pillow)
notes/
  <id>.json              — One file per note: {id, title, content, tags, created, modified}
  uploads/               — Uploaded images (covered by existing volume mount)
```

## Architecture
- **No framework**: vanilla JS with `async/await`, no bundler
- **Editor**: EasyMDE with `inputStyle: 'contenteditable'` (CodeMirror 5 underneath)
- **Rendering**: server-side via `/api/render` (Go goldmark), highlight.js client-side
- **Storage**: flat JSON files, notes loaded into `let notes = []` array in memory

## Key state (app.js)
| Variable | Purpose |
|---|---|
| `currentId` | ID of the open note, `null` on welcome screen |
| `isEditing` | Whether editor is shown vs rendered view |
| `_viewSeq` | Incremented to abort stale `showViewMode()` renders |
| `_selectSeq` | Incremented to abort stale `selectNote()` calls |
| `tagsOpen` | Whether the tags sidebar section is expanded |
| `calendarOpen` | Whether the calendar sidebar section is expanded |

## Important implementation decisions

### Edit delay fix
`easyMDE.value()` is called in `showEditMode()` (not `selectNote()`). Loading content into CodeMirror while its container has `display:none` forces an expensive re-render when shown. Always populate the editor only after the container is visible.

### Slash command palette keys
Arrow keys and Enter in the slash command palette use a DOM **capture-phase** listener on the CodeMirror wrapper element — NOT `cm.addKeyMap()`. CM5's addKeyMap with `return false` does not reliably fall through to downstream handlers.

### Gradient text fix
When overriding `background` on an element with `-webkit-background-clip: text`, you **must** re-declare all three properties or the gradient renders as an opaque rectangle:
```css
background: ...;
-webkit-background-clip: text;
-webkit-text-fill-color: transparent;
background-clip: text;
```

### Race condition guard
`showViewMode()` snapshots `const seq = ++_viewSeq` and `const id = currentId` at entry, then checks `if (seq !== _viewSeq || currentId !== id) return` after every `await`.

### Welcome screen
Shown when `currentId === null`. Navigated to via `showWelcomeScreen()`. Deleting a note always calls `showWelcomeScreen()`. Brand icon/name in sidebar also navigates to it.

## Removed capabilities
Mermaid diagrams and KaTeX math were removed (caused render race conditions and added load weight). The `SLASH_COMMANDS` array and Guide modal no longer include `/diagram` or `/math`.

## Features
- Markdown editing (EasyMDE / CodeMirror 5)
- Server-side rendering (goldmark)
- Syntax highlighting (highlight.js, 60+ languages)
- Wiki-style `[[Note Title]]` links with backlinks panel
- Tags with colour coding and sidebar filter
- Collapsible calendar and tags sections in sidebar
- Note graph (force-directed, canvas-based)
- Image upload (toolbar, drag-and-drop, paste, or `/image` slash command) → stored in `notes/uploads/`
- Slash command palette (`/h1`, `/h2`, `/todo`, `/table`, `/link`, `/image`)
- Auto-save (1.5 s debounce)
- Export to `.md`, `.html`, `.pdf`
- Table of contents (auto-generated for notes with 2+ headings)
- Dark/light theme toggle
- Mobile-responsive with sidebar drawer
- Particle animation on welcome screen
